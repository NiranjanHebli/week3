# Day 16 - AM - Hypothesis Testing & Confidence Intervals


## Part A:- Concept Application 

This task involves designing a hypothesis test to evaluate whether a new checkout-page design increases average order value in an e-commerce platform. We compare two independent customer groups using an independent two-sample t-test.


## Key Learnings

- A hypothesis test converts a business question into measurable statistical evidence.

- Choosing between one-tailed and two-tailed testing depends on the business objective being tested.

- The independent t-test is appropriate when comparing means of two unrelated groups.

- Statistical significance (p-value) and practical significance (effect size) should both be reported.

- Confidence intervals help estimate the likely range of true business impact, not just whether an effect exists.

## Part B :- Stretch Problem 

The multiple comparison problem arises when many hypothesis tests are performed simultaneously, increasing the probability of obtaining statistically significant results purely by chance. To control this inflated error, Bonferroni correction divides α by the number of tests, producing a stricter threshold

## Key Learning;-

- Multiple testing greatly increases the chance of false discoveries even when each individual test uses α = 0.05.

- Simulation helps verify theoretical probability and demonstrates how often random significance appears.

- Bonferroni correction controls family wise error by lowering the significance threshold for each test.
